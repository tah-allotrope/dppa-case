03 / Math / Algorithm Used

Math / Algorithm Used

Material math used in the phase

The core algorithm computes hourly load-versus-generation overlap and then derives settlement components from those interval volumes.

  • Q_match[t] = min(load, generation)
  • Q_shortfall[t] = max(load - generation, 0)
  • Q_excess[t] = max(generation - load, 0)
  • EVN = matched x market x loss + matched x DPPA + shortfall x retail
  • Developer = contract quantity x (strike - market)

The app also calculates a simplified matched-kWh intuition:

  • matched kWh ~= strike + DPPA charge + loss adjustment
  • This is only surfaced as the headline shortcut when contract quantity tracks matched consumption.
  • When contract quantity exceeds matched consumption, the app raises an explicit warning because the simplification breaks.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

Chosen path limitations

  • The MVP uses hourly intervals and flat pricing assumptions; it is a teaching model, not an invoice engine.
  • The "allocated generation" mode uses a heuristic 88% factor to create a realistic-feeling intermediate case, not a regulatory allocation engine.
  • The current app is local-first; Firebase deployment was intentionally deferred so the finance narrative could be proven first.

Second-best alternative

Streamlit was the second-best path because it would have made Python-side financial modeling easy, but it lost to the static Vite app on visual polish, local demo smoothness, and future Firebase Hosting compatibility.

07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Patch collision: the first bulk patch removed src/main.js after re-adding it; resolved by restoring the intended entrypoint and re-reading the core files before verification.
  • Test expectation drift: the first EVN total assertion incorrectly divided by 24; resolved by tightening the test to the single-interval example values from the buyer report.
  • Browser console noise: the app served cleanly but produced a favicon.ico 404; resolved by adding app/public/favicon.ico so the demo has no distracting runtime error.
